❓ How to Use Moxie Project Plans
Quick start: Click ＋ New Project → fill in the wizard → you're ready to go. Everything auto-saves as you work.
📂 Managing Projects
＋ New ProjectStart a fresh project — opens a setup wizard for name, due date, lead, and stakeholders 📂 My ProjectsSee all your saved projects and switch between them ⬇ ExportDownload a .json backup of your project (save a copy outside the browser) ⬆ LoadRestore a project from a previously exported .json file ✕ ClearRemove the current project from this browser (does not delete Firebase data)
🗂 The Tabs
📊 DashboardLive summary — task counts, stakeholder status, flagged risks, upcoming due dates ⚙️ Project SetupEdit project details, due date, lead, and which teams are included or have had the plan shared 📄 Proposal BuilderFill out the form to generate a formatted project proposal — copy to Google Docs with one click ✅ Tasks & DeliverablesAdd and manage all tasks — set owner, due date, and status. Load template packs or import from a doc 🚀 Launch ChecklistPre-built go-live checklist — check off each item, view examples, add notes, copy templates 📝 Meeting NotesLog meetings — paste a transcript and get auto-drafted notes and action items ⚠️ Risk RegisterFlag risks from tasks or add manually — tracks owner, severity, and mitigation plan 📋 Digests & BriefsGenerate a weekly digest, daily standup, or pre-meeting brief to share with stakeholders
🔄 Saving & Syncing
✓ Saved badgeYour project auto-saves to your browser every 2 seconds after any change — no manual save needed ☁️ Live SyncConnect Firebase to sync your project in real time with anyone who has your share link 🔗 Share LinkOnce Firebase is connected, copy your project's unique link — anyone who opens it sees the same live project ● Live badgeGreen dot means Firebase is active and syncing changes across all users in real time
👥 Working as a Team
  • Each PM goes to newprojectplanops.netlify.app and clicks ＋ New Project to create their own project
  • Each project gets its own unique ID — no two PMs share data unless they intentionally share a link
  • To invite someone to view or edit your project: click 🔗 Share Link → send them that URL
  • Anyone with the link opens it in their browser and sees your live project instantly — no login needed
  • Use 📧 Send to Team to email a status update to all stakeholders listed in your project

How it works: Firebase stores your project in Google's cloud (free). Everyone with the share link opens the same live data — changes sync within seconds. No login, no server, works anywhere.
1
Create a free Firebase project

Go to console.firebase.google.com → Add project → name it (e.g. moxie-projects) → Continue → Disable Google Analytics → Create project.

2
Add a Web App & copy the config

In your Firebase project → click the </> Web icon → register app (name it anything) → copy the firebaseConfig object shown.

3
Enable Realtime Database

In Firebase console → Build → Realtime Database → Create database → Start in test mode → Enable. Keep the default region.

4
Paste your Firebase config below

Paste the entire config object (looks like the example below). This is stored only in your browser.

Setup takes ~5 minutes. Free Firebase tier is more than enough for this use case.
