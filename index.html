❓ How to Use Moxie Project Plans
Quick start: Click ＋ New Project → fill in the wizard → you're ready to go. Everything auto-saves and syncs to your team in real time.
📂 Managing Projects
＋ New ProjectStart a fresh project — wizard captures name, due date, lead, and stakeholders. 4 default tasks auto-add: Demo env, Question flow, Post-launch handoff, Churn process. 📂 My ProjectsSee all saved projects and switch between them ⬇ ExportDownload a .json backup (save outside the browser as a safety net) ⬆ LoadRestore a project from a previously exported .json file 📧 Send to TeamEmail project status to all included stakeholders (uses POC emails from Stakeholder Plan)
🗂 The Tabs (in order)
📊 DashboardLive summary — RAG status, task counts, flagged risks, upcoming due dates, project owners by team. "✅ Complete Project →" button opens the launch checklist + handoff modal. 🗂 PlanTasks + action items grouped by phase. Inline edit owner, due date, priority, status. "✅ Complete Project →" button (top-right) opens the project signoff modal with Launch Checklist + Handoff & SOPs. 🧭 WorkstreamsTasks + AIs grouped by TEAM. Owner is enforced as a team label only — clean roll-up by Sales, PSM, Onboarding, etc. 📅 CalendarSchedule meetings with Google Calendar — auto-Meet link, recurring options, POC checkboxes. Import existing events. 🤖 AI AssistantHub for all AI features — 5 sub-tabs: Ask & Health · Reports & Briefs · Drafts & Comms · Smart Tools · Settings. See details below. ⚠️ Risk RegisterFlag risks. Tracks owner, likelihood, impact, mitigation. 🔗 LinksQuick reference URLs — paste a link, auto-pulls page title. ⚙️ Project SetupProject details + Stakeholder Plan (14 teams). 🔒 Pre-Launch Checklist at top — 3 required items (Proposal, Stakeholder Alignment, Plan Created/Shared) that must be cleared before work begins. 📋 DecisionsLog key decisions with context, owner, date. Auto-populates from AI transcript processing. 💰 BudgetBudget total, contingency, line items by category. 🧩 Process FlowLucidchart/Miro/Figma links + inline Mermaid editor.
Removed nav tabs (features moved):
  • 🚀 Launch Checklist + 🤝 Handoff & SOPs → both now inside the "✅ Complete Project" modal (button on Plan or Dashboard)
  • 📧 Digests & Briefs → moved to AI Assistant > Reports & Briefs
  • 📄 Proposal Builder → moved to AI Assistant > Drafts & Comms (AI-powered, modeled on Moxie's template)
🔒 Pre-Launch Pattern: When you open the app, if any of the 3 pre-launch items are still unchecked, a popup nudges you to finish them on the Project Setup tab. Project work shouldn't begin until pre-launch is cleared.

🤝 Handoff & SOPs
  • Capture the new owner taking over post-launch — name, role, email, effective date
  • Track 5-step handoff checklist with auto-captured signoff dates from PM and new owner
  • Build SOPs with status (Draft → Reviewed → Signed Off), owner, external link, content
  • Use 📋 Insert SOP Template for a pre-structured template (Purpose / Scope / Procedure / Exceptions / Quality Checks)
🔄 Saving & Syncing
✓ Saved badgeAuto-saves every 2 seconds to your browser AND to Firebase (cloud) ☁️ Live SyncFirebase is pre-configured — any change syncs to teammates within ~2 seconds 🔗 Share LinkCopy your project's URL (with #proj_xxx) — anyone who opens it sees the same live project ● Live badgeGreen dot in header = Firebase active and syncing changes across users in real time
👥 Working as a Team
  • Each PM creates their own project — gets a unique #proj_xxx URL
  • Share a project by sending its full URL (with the hash) to a teammate
  • No login needed — anyone with the link opens it and sees the live project instantly
  • Edits from any teammate sync to all others within ~2 seconds via Firebase
  • Use 📧 Send to Team or AI Assistant Send Status to email all stakeholder POCs
✅ Task Status Options
Not Started In Progress At Risk Done Blocked On Hold
Marking a task At Risk or Blocked auto-copies it to the Risk Register tab.
📌 Default Tasks (Auto-Added to Every Project)
  • Demo environment created (Product) — Confirm a working demo is provisioned
  • Define flow for where future questions go (Ops) — Routing process for new questions
  • Document post-launch question flow & service handoff (Ops/Product) — Service handoff path
  • Define churn / cancellation & off-boarding process (Ops/Billing) — When customers leave: billing stop, data archival, refunds
💡 Tip: Bookmark your project's #proj_xxx URL so you can come back to it directly. Editing a task on the Plan tab moves it between phases via the Phase dropdown — you'll be prompted for a "Move Reason" which is logged to the task notes.

            Enable Realtime Database
            
In Firebase console → Build → Realtime Database → Create database → Start in test mode → Enable. Keep the default region.
